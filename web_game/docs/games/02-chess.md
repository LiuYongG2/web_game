# ♟️ 中国象棋 — 游戏架构文档

## 概述
完整实现中国象棋双人对战，含全部走法规则、将军/将死/困毙检测、悔棋、走棋记录。

## 技术栈
- 渲染: Canvas 2D API (棋盘+棋子全部代码绘制)
- 音效: Web Audio API

## 架构

```
index.html
│
├── Canvas渲染层
│   └── draw()
│       ├── 棋盘: 木纹渐变底色 + 网格线 + 楚河汉界文字 + 九宫斜线
│       ├── 上步高亮: 起止位置金色半透明圆
│       ├── 选中高亮: 金色描边圆
│       ├── 合法落点: 绿色小圆(空格) / 红色虚线圆(吃子)
│       └── 棋子: 径向渐变底色 + 色环 + 高光椭圆 + 楷体汉字
│
├── 数据模型
│   ├── board[10][9] — 二维数组, 每格 null 或 {name, color}
│   ├── currentTurn — 'red' | 'black'
│   ├── selected — {col, row} | null
│   ├── validMoves[] — [[col,row], ...]
│   ├── moveHistory[] — {from, to, piece, captured}
│   └── lastMove — 上一步起止位置 (用于高亮)
│
├── 规则引擎 (核心)
│   ├── getValidMoves(col, row) — 计算合法走法
│   │   ├── 7种棋子各自走法逻辑
│   │   └── 过滤自杀走法 (模拟走棋→检查是否被将)
│   │
│   ├── getRawMoves(col, row) — 不含自杀过滤的原始走法
│   │
│   ├── isInCheck(color) — 将军检测
│   │   ├── 遍历对方所有棋子攻击范围
│   │   └── 额外: 将帅对面检测 (同列无遮挡)
│   │
│   ├── isCheckmate(color) — 将死/困毙检测
│   │   └── 遍历己方全部棋子, 无合法走法 = 败
│   │
│   └── findKing(color) — 查找将/帅位置
│
└── 走棋记录
    └── logMove() — 追加到滚动面板, 中文纵列名
```

## 关键技术点

### 1. 七种棋子走法规则

| 棋子 | 移动规则 | 特殊限制 |
|------|---------|---------|
| 車 | 直线任意距离 | 不可越子 |
| 馬 | 日字形 (2+1) | 蹩马腿: 检查中间阻挡格 |
| 象/相 | 田字形 (2+2) | 塞象眼 + 不可过河 |
| 士/仕 | 斜线一格 | 限九宫 (3≤col≤5) |
| 將/帥 | 直线一格 | 限九宫 + 不可对面 |
| 炮/砲 | 移动同車 | 吃子: 必须隔一子跳吃 |
| 兵/卒 | 前进一格 | 过河后可左右 |

### 2. 将军检测算法
`isInCheck(color)` 执行两步:
1. 遍历对方全部棋子，调用 `getRawMoves()` 获取攻击范围，检查是否覆盖己方将/帅位置
2. 特殊规则: 检查两个将帅是否在同一列且中间无棋子 (对面笑)

### 3. 走子合法性验证
`getValidMoves()` 在生成所有可能走法后，对每个走法执行模拟:
```
保存棋盘 → 执行走法 → 检查isInCheck(己方) → 恢复棋盘
```
过滤掉会导致自己被将的走法，确保每个返回的走法都是合法的。

### 4. 3D 棋子渲染
使用 `createRadialGradient()` 创建从高光到阴影的径向渐变，模拟棋子的立体感:
- 外层: 实心底色圆 (#fff5e0 → #d4b896)
- 中层: 颜色圆环描边 (红/黑)
- 顶层: 半透明白色椭圆高光
- 文字: 楷体汉字居中
