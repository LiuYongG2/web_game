# 🔍 捉迷藏 — 游戏架构文档

## 概述
隐藏物件寻找游戏。3个程序化绘制的场景中隐藏着emoji物品，玩家凭观察力找到全部物品。

## 技术栈
- 渲染: Canvas 2D API (程序化场景绘制)
- 音效: Web Audio API

## 架构

```
index.html
│
├── Canvas渲染
│   └── drawScene()
│       ├── 调用当前关卡的 draw(ctx, w, h) 绘制完整场景
│       ├── 绘制已找到物品的绿色标记环
│       ├── 绘制隐藏物品 (18px emoji, globalAlpha=0.65)
│       └── 绘制发光动画效果 (glowFx[])
│
├── 数据模型
│   ├── LEVELS[3] — 关卡数组 {name, items[], draw()}
│   │   每个item: {emoji, name, x, y}
│   ├── foundItems[] — 已找到物品名称
│   ├── misses — 错误点击计数
│   └── glowFx[] — 活跃发光动画
│
├── 场景绘制系统
│   ├── 厨房 — 冰箱、灶台、烤箱、台面、灯、盘子 (6个隐藏物)
│   ├── 花园 — 树木、花朵、栅栏、池塘、长椅 (7个隐藏物)
│   └── 书房 — 书架、书桌、台灯、地球仪 (7个隐藏物)
│
├── 点击检测
│   └── handleClick(px, py)
│       ├── 遍历未找到物品
│       ├── 欧氏距离 < 30px → 命中
│       └── 无命中 → misses++
│
└── 计分系统
    └── 物品分(50/个) + 时间奖励(300-秒×2) + 准确奖励(100-错误×10)
```

## 关键技术点

### 1. 程序化场景绘制
每个场景使用20-30次Canvas绘图调用组合画面。核心API:
- `fillRect()` — 矩形物体 (家具、墙面)
- `arc()/ellipse()` — 圆形物体 (盘子、池塘)
- `createLinearGradient()` — 天空、地面渐变
- 像素噪点 — 循环随机fillRect模拟纹理质感

### 2. 物品隐藏策略
物品以小号emoji (18px) 绘制，`globalAlpha=0.65` 降低透明度使其半融入背景。每个物品有固定坐标和30px点击判定半径。

### 3. 发光环动画系统
`glowFx[]` 数组管理所有活跃的动画效果:
- 命中: 绿色扩散环 (1200ms)
- 未命中: 红色✕标记 (1200ms)
- 每帧根据存活时间计算透明度和尺寸
- 超期自动从数组移除
- 有动画时启动 `requestAnimationFrame` 循环
