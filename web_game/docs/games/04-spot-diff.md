# 🧐 找不同 — 游戏架构文档

## 概述
双图找茬游戏。左右两张Canvas程序化绘制的相似图片，每关有5处差异需要发现。

## 技术栈
- 渲染: Canvas 2D API (双Canvas同步绘制)
- 音效: Web Audio API

## 架构

```
index.html
│
├── 双Canvas系统
│   ├── canvasL (左图) ← drawBase() 绘制原图
│   ├── canvasR (右图) ← drawDiff() 绘制差异图
│   └── 两个Canvas共享同一个 handleClick() 处理函数
│
├── 数据模型
│   ├── LEVELS[3] — {name, diffs[], drawBase(), drawDiff()}
│   │   每处差异: {x, y, r(判定半径)}
│   ├── foundDiffs[] — 已找到差异的索引
│   └── hints — 剩余提示次数 (初始3)
│
├── 差异实现
│   └── drawDiff() 调用 drawBase() 后:
│       ├── 用背景色覆盖差异区域
│       └── 重新绘制变体 (改颜色/删除/添加/改方向)
│
├── 提示系统
│   └── useHint()
│       ├── 随机选取未找到差异
│       ├── setInterval 250ms 闪烁 (橙色环, 8次)
│       ├── 两个Canvas同步绘制高亮
│       └── 扣30分
│
└── 3个关卡场景
    ├── 城市街景 — 汽车颜色/缺窗户/招牌文字/多树/云朵形状
    ├── 海底世界 — 缺鱼/珊瑚色/多气泡/宝箱/缺海星
    └── 太空站 — 星群/行星色/火箭向/天线/缺外星人
```

## 关键技术点

### 1. 双Canvas同步
两个Canvas各 400×350，并排显示。`drawBoth()` 分别调用 `drawBase()` 和 `drawDiff()`，然后在两图的相同坐标位置绘制已发现差异的绿色标记环。

### 2. 差异绘制方法
`drawDiff()` 先调用 `drawBase()` 绘制完整原图，然后对每处差异:
1. 用背景色 `fillRect()` 覆盖原有区域
2. 在相同位置绘制变体版本
3. 差异类型: 颜色变化、元素删除、元素添加、方向翻转

### 3. 点击判定
两个Canvas注册相同的 `handleClick` 函数。遍历未找到差异，计算点击点与差异中心的欧氏距离，小于 `r+5` 像素即命中。
