# MyWeb 游戏平台 — 技术文档与使用说明

## 一、项目概述

MyWeb 游戏平台是一个基于 Node.js 的全栈 Web 应用，集成了 6 款风格各异的浏览器小游戏。整个项目不依赖前端框架（无 React/Vue），全部游戏采用原生 HTML + CSS + JavaScript 实现，图形渲染使用 Canvas 2D API（FPS 游戏使用 Three.js 3D 引擎），音效通过 Web Audio API 程序化生成，无需任何外部图片或音频文件。

## 三、服务器搭建详解

### 3.1 初始化过程

项目通过以下步骤搭建：

```bash
# 1. 创建项目目录
mkdir -p ~/下载/micode/myweb && cd ~/下载/micode/myweb

# 2. 初始化 npm 项目
npm init -y

# 3. 安装核心依赖
npm install express ejs
```

仅需 2 个依赖包：`express`（Web 框架）和 `ejs`（模板引擎）。

### 3.2 Express 服务器配置

`server.js` 是整个应用的入口，共 35 行代码，负责以下核心功能：

**模板引擎绑定**：将 EJS 设置为默认视图引擎，模板文件存放在 `views/` 目录。Express 收到请求后，通过 `res.render()` 将 EJS 模板编译为 HTML 返回给浏览器。

**静态文件托管**：通过 4 条 `express.static()` 中间件，将不同目录映射到不同 URL 路径：

| 目录 | URL 路径 | 说明 |
|------|----------|------|
| `public/` | `/` | 平台 CSS/JS（与根路径合并） |
| `assets/` | `/assets/` | 公共样式和预留资源 |
| `shared/` | `/shared/` | 游戏共享工具库 |
| `games/` | `/games/` | 全部游戏的 HTML 文件 |

**请求体解析**：通过 `express.json()` 和 `express.urlencoded()` 中间件，支持 JSON 和表单格式的 POST 请求体解析，为后续 API 扩展预留能力。

**路由定义**：

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 渲染游戏平台主页（EJS 模板） |
| `/about` | GET | 渲染关于页面 |
| `/api/hello` | GET | JSON API 示例接口 |
| `*`（兜底） | ALL | 返回 404 页面 |

**关键设计决策**：游戏文件全部作为静态 HTML 托管，不经过模板引擎。这意味着每个游戏是完全独立的单文件应用，可以脱离服务器在浏览器中直接打开。服务器仅负责文件分发，不参与游戏逻辑。

### 3.3 两种启动模式

```bash
npm start   # 生产模式：node server.js
npm run dev # 开发模式：node --watch server.js（文件变动自动重启）
```

`--watch` 是 Node.js v18+ 内置的文件监听功能，无需安装 nodemon 等第三方工具。

## 四、平台主页实现

### 4.1 视觉设计

主页采用暗色系宇宙风格，使用 CSS 线性渐变（`linear-gradient(135deg, #0f0c29, #302b63, #24243e)`）作为背景。标题文字使用三色渐变填充（`background-clip: text`），呈现从粉紫到金色的流光效果。

### 4.2 游戏卡片网格

使用 CSS Grid 布局（`grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))`）实现自适应网格。每张卡片包含三个区域：

- **图标区**：大号 Emoji 配合渐变背景，每款游戏有独立的配色方案
- **信息区**：游戏名称和简介
- **底部栏**：分类标签（益智/策略/休闲/动作）+ 高分记录

卡片悬停时有 `translateY(-6px)` 上浮动画和投影增强，提供操作反馈。

### 4.3 高分系统

页面加载时，通过 JavaScript 遍历所有 `.high-score` 元素，根据 `data-game` 属性从 `localStorage` 读取对应游戏的最高分并显示。localStorage 的 key 命名规则为 `highscore_{游戏标识}`，如 `highscore_goose_game`。

## 五、游戏开发详解

### 5.1 共享基础设施

#### GameTemplate.js — 游戏基类

提供所有游戏通用的生命周期管理：

- **状态管理**：`isRunning`、`isPaused` 状态标记，支持暂停/恢复
- **Canvas 工厂**：`createCanvas()` 方法创建自适应尺寸的画布
- **倒计时系统**：`startTimer()` 基于 `setInterval` 实现秒级倒计时
- **计分系统**：`addScore()` 累加分数并触发回调
- **高分持久化**：`saveHighScore()` / `loadHighScore()` 封装 localStorage 读写
- **资源清理**：`destroy()` 方法清除定时器和 DOM 元素

#### utils.js — 工具函数库

| 函数 | 用途 |
|------|------|
| `randomInt(min, max)` | 生成指定范围的随机整数 |
| `shuffle(array)` | Fisher-Yates 洗牌算法，返回新数组 |
| `formatTime(seconds)` | 秒数格式化为 `MM:SS` |
| `playSound(freq, dur, type)` | Web Audio API 音效生成 |
| `soundEffects.*` | 预置音效：点击、消除、胜利、失败、射击、命中 |
| `drawRoundRect()` | Canvas 圆角矩形绘制辅助 |

#### 音效生成原理

所有音效通过 Web Audio API 实时合成，无需加载音频文件：

```
AudioContext → OscillatorNode（振荡器）→ GainNode（音量控制）→ 输出
```

通过控制振荡器的频率（Hz）、波形类型（sine/square/sawtooth）和音量包络线（exponentialRampToValueAtTime 实现衰减），组合出不同的音效。例如"消除成功"音效是三个递增频率（523Hz→659Hz→784Hz）的正弦波依次播放，形成上行音阶。

### 5.2 🦆 抓大鹅（Goose Game）

**文件**：`games/goose-game/index.html`（394行）
**类型**：三消堆叠类 · Canvas 2D

#### 核心机制

这是一款类"羊了个羊"的三消堆叠游戏。场景中有 3 层互相遮挡的方块，玩家点击未被遮挡的方块将其移入底部槽位（最多 7 个），当槽位中出现 3 个相同图案时自动消除。所有方块清除即过关，槽位填满则失败。

#### 关键技术

**三层遮挡系统**：方块按 `layer` 属性分为 3 层，每层有 12px 的偏移量产生层叠视觉效果。点击检测时从高层向低层遍历（`for (let i = tiles.length - 1; ...)`），高层方块优先响应。`isTileBlocked()` 函数通过矩形碰撞检测（AABB）判断某方块是否被更高层的方块遮挡。

**可解性保证**：生成方块时，先创建成对的 Emoji 组（每种 3 个为一组），确保理论上总能完全消除。然后通过 Fisher-Yates 洗牌算法随机分布到各层。

**关卡设计**：

| 关卡 | 图案种类 | 方块总数 | 网格尺寸 |
|------|----------|----------|----------|
| 1 | 6 种 | 36 | 6×6 |
| 2 | 8 种 | 48 | 8×6 |
| 3 | 10 种 | 60 | 8×8 |

**道具系统**：每关各 1 次使用机会
- 移出：移除槽位最后一个方块
- 撤回：恢复上一步操作（利用 `history` 栈记录）
- 洗牌：随机重新分配剩余方块的图案

**Emoji 渲染**：使用 `ctx.fillText()` 将 Emoji 直接绘制到 Canvas 上，利用系统字体渲染彩色图标，无需加载图片资源。

### 5.3 ♟️ 中国象棋（Chinese Chess）

**文件**：`games/chess/index.html`（525行）
**类型**：策略棋类 · Canvas 2D

#### 核心机制

完整实现中国象棋（象棋/Xiangqi）的双人对战模式。包含 9×10 标准棋盘、32 颗棋子、全部走法规则、将军/将死/困毙检测，以及悔棋功能。

#### 关键技术

**棋盘绘制**：Canvas 绘制传统木色棋盘（`#d4a76a`），包含：
- 9 条竖线、10 条横线（中间断开形成楚河汉界）
- 两个九宫格的对角斜线
- "楚河""漢界"文字
- 外框加粗边线

**棋子绘制**：每颗棋子由三层构成：
1. 实心圆底（`#f5e6c8`）
2. 色环（红方 `#c0392b` / 黑方 `#2c3e50`）
3. 楷体汉字（红方：帥仕相車馬炮兵 / 黑方：將士象車馬砲卒）

**走法规则引擎**：`getValidMoves(col, row)` 函数实现了全部 7 种棋子的移动规则：

| 棋子 | 规则要点 |
|------|----------|
| 車 | 直线移动，不可越子 |
| 馬 | 日字形移动，有蹩马腿检测 |
| 象/相 | 田字形移动，有塞象眼检测，不可过河 |
| 士/仕 | 斜线移动一格，限九宫内 |
| 將/帥 | 直线移动一格，限九宫内 |
| 炮/砲 | 直线移动不可越子，吃子必须隔一子跳吃 |
| 兵/卒 | 过河前只能前进，过河后可左右移动 |

**将军检测**：`isInCheck(color)` 遍历对方所有棋子的攻击范围，判断己方将/帅是否在威胁中。额外实现了"对面笑"规则（两个将帅不可在同一列无遮挡直线对面）。

**走子合法性验证**：每次生成合法走法后，模拟执行该走法并检测自己是否被将军，过滤掉会导致自杀的走法。

**将死/困毙判定**：`isCheckmate(color)` 遍历当前方所有棋子，若没有任何合法走法则判定为输。结合 `isInCheck()` 区分将死（被将军且无解）和困毙（未被将军但无子可动）。

**悔棋系统**：`moveHistory` 数组记录每步的起止位置和被吃棋子，`undoMove()` 弹出最后一条记录恢复棋局。

### 5.4 🔍 捉迷藏（Hide & Seek）

**文件**：`games/hide-seek/index.html`（417行）
**类型**：隐藏物件寻找 · Canvas 2D

#### 核心机制

每个关卡提供一幅 Canvas 程序化绘制的场景图，场景中隐藏着 6-7 个小物品（Emoji 图标）。玩家需要在场景中找到所有隐藏物品，根据用时和准确率计分。

#### 关键技术

**程序化场景绘制**：3 个场景完全通过 Canvas API 代码绘制，无需外部图片：

| 关卡 | 场景 | 绘制内容 | 隐藏物品数 |
|------|------|----------|------------|
| 1 | 厨房 | 冰箱、灶台、烤箱、吊灯、盘子、餐柜 | 6 |
| 2 | 花园 | 树木、花朵、栅栏、池塘、长椅、木箱 | 7 |
| 3 | 书房 | 书架、书桌、台灯、地球仪、电脑、落地灯 | 7 |

每个场景使用 20-30 次 Canvas 绘图调用（`fillRect`、`arc`、`ellipse`、渐变填充等）组合出完整画面。

**物品隐藏策略**：隐藏物品以小尺寸 Emoji（18px）绘制在场景中，通过 `globalAlpha = 0.7` 降低透明度使其融入背景。每个物品有固定的坐标和 28px 的点击判定半径。

**点击反馈系统**：
- 正确点击：绿色发光环动画 + 双音阶上升音效 + 浮动文字提示
- 错误点击：红色 ✕ 标记动画 + 低沉音效

发光环通过 `glowEffects` 数组管理，每帧根据存活时间绘制扩散和淡出效果，过期后自动移除。

**计分公式**：`总分 = 找到物品分(50×个数) + 时间奖励(max(0, 300 - 秒数×2)) + 准确奖励(max(0, 100 - 错误次数×10))`

### 5.5 🧐 找不同（Spot the Difference）

**文件**：`games/spot-diff/index.html`（353行）
**类型**：双图找茬 · Canvas 2D

#### 核心机制

左右并排显示两张相似但有细微差异的图片（均为 Canvas 程序化绘制），玩家点击任意一张图中的差异点即可标记。提供提示功能和计时系统。

#### 关键技术

**双画布同步绘制**：使用两个独立的 Canvas 元素（`canvasL` 和 `canvasR`），分别调用 `drawBase()` 和 `drawDiffVersion()` 方法。两个方法共享基础绘制代码，`drawDiffVersion()` 在基础之上叠加差异绘制。

**差异实现方式**：每个差异通过"先覆盖原区域再重新绘制"实现。例如城市街景关卡中的"汽车颜色不同"：右图先用背景色覆盖原蓝色汽车区域，再绘制一辆红色汽车。

**3 个关卡场景**：

| 关卡 | 场景 | 差异内容示例 |
|------|------|-------------|
| 1 | 城市街景 | 汽车颜色、缺少窗户、招牌文字、多一棵树、云朵形状 |
| 2 | 海底世界 | 缺少鱼、珊瑚颜色、多出气泡、宝箱不同、缺少海星 |
| 3 | 太空站 | 星星群缺少、行星颜色、火箭方向、多一根天线、缺少外星人 |

**提示系统**：每关 3 次提示机会。使用提示会启动一个 `setInterval` 闪烁动画（250ms 间隔、8 次闪烁），在两张画布上同步绘制橙色高亮环。每次使用提示扣 30 分。

**双画布同步标记**：找到差异时，在两个 Canvas 上的相同坐标位置同时绘制绿色圆环标记，通过 `foundDiffs` 数组追踪已发现的差异索引。

### 5.6 🍬 开心消消乐（Match-3）

**文件**：`games/match-game/index.html`（353行）
**类型**：三消益智 · Canvas 2D

#### 核心机制

经典的 Bejeweled/Candy Crush 风格三消游戏。8×8 网格中排列着 6 种水果 Emoji，交换相邻方块使 3 个或更多相同方块排成一行/列即可消除，消除后上方方块下落填补空缺，可触发连锁消除。

#### 关键技术

**消除检测算法**：`findMatches()` 函数分两趟扫描：
1. 横向扫描：逐行检查连续 3+ 个相同方块
2. 纵向扫描：逐列检查连续 3+ 个相同方块

使用 `Set` 去重（同一方块可能同时属于横向和纵向匹配）。

**重力掉落系统**：`applyGravity()` 对每列从底部向上扫描，将非空方块"压实"到底部，空出的顶部位置填入随机新方块。

**连锁消除**：`processMatches()` 使用 `async/await` 配合 `sleep()` 实现动画节奏：
```
检测匹配 → 消除(200ms延迟) → 重力下落(200ms延迟) → 再次检测 → 循环
```
每次循环 `comboCount` 递增，分数乘以 `(1 + combo × 0.5)` 的倍率。

**初始棋盘无匹配保证**：生成棋盘时，每放置一个方块都调用 `hasMatchAt(r, c)` 检查是否与左边两个或上面两个相同，如果匹配则重新随机，确保初始状态无可消除组合。

**死局检测与洗牌**：每次消除结算后，`hasValidMoves()` 遍历所有相邻交换，模拟执行后检查是否产生匹配。若无有效走法，自动调用 `shuffleBoard()` 重排棋盘。

**关卡系统**：

| 关卡 | 目标分数 | 步数限制 |
|------|----------|----------|
| 1 | 500 | 30 |
| 2 | 1500 | 30 |
| 3 | 3000 | 30 |

### 5.7 🔫 FPS 射击（First-Person Shooter）

**文件**：`games/fps-game/index.html`（622行）
**类型**：第一人称射击 · Three.js 3D

这是整个平台中技术复杂度最高的游戏，也是唯一使用 3D 引擎的游戏。

#### 核心机制

玩家处于一个室内射击场中，通过鼠标控制视角，射击出现的靶标和敌人。游戏持续 60 秒，期间需要管理弹药、切换武器、躲避敌人接近。

#### 关键技术

**Three.js 场景构建**：使用 ES Module 方式从 unpkg CDN 加载 Three.js。场景包含：
- 地板/天花板：`PlaneGeometry` + `MeshStandardMaterial`
- 四面墙壁：4 个旋转的 `PlaneGeometry`
- 光照：环境光（`AmbientLight`） + 方向光（`DirectionalLight`） + 点光源（`PointLight`）
- 雾效：`Fog` 增加空间深度感

全部 3D 模型使用 Three.js 内置几何体组合，无外部模型文件。

**FPS 视角控制**：使用 Pointer Lock API（`requestPointerLock()`）锁定鼠标指针，通过 `mousemove` 事件的 `movementX/Y` 控制摄像机的 yaw（水平旋转）和 pitch（俯仰角），pitch 限制在 ±85° 防止翻转。摄像机旋转顺序设为 `'YXZ'` 以正确处理欧拉角。

**射线检测命中判定**：每次射击创建 `Raycaster`，从摄像机位置沿视线方向发射射线，通过 `intersectObjects()` 检测与靶标/敌人的碰撞。霰弹枪则同时发射 5 条带随机偏移的射线模拟散射效果。

**爆头判定**：命中点的 Y 坐标高于目标中心 0.2 单位以上判定为爆头，奖励 25 分（普通 10 分）。

**三种武器系统**：

| 武器 | 弹夹 | 储备 | 射速 | 特性 |
|------|------|------|------|------|
| Pistol (1) | 12 | ∞ | 400ms | 单发精准 |
| Rifle (2) | 30 | 90 | 120ms | 连射，轻微散布 |
| Shotgun (3) | 8 | 24 | 800ms | 5 射线散射 |

按数字键 1/2/3 切换，R 键换弹（1.5 秒动画）。弹夹耗尽自动提示换弹。

**靶标系统**：3 种靶标 + 1 种惩罚靶标：
- 静态靶标：固定位置的红白靶
- 移动靶标：沿 X 轴左右滑动（速度 1-2 单位/秒）
- 旋转靶标：绕 Y 轴自转
- 骷髅靶标（黑色球体 + 红色发光眼睛）：命中扣 15 分

每 5 秒生成一波新靶标（3-5 个 + 30% 概率出骷髅靶）。

**敌人 AI**：3 个敌人由方块组合成简易人形（身体+头部+双臂）。行为逻辑：
- 始终朝玩家直线移动（速度 0.5-1 单位/秒）
- 受击闪红（`emissive` 材质属性 150ms 闪烁）
- 2 次命中后死亡，掉落弹药拾取物（金色旋转方块）
- 死亡 10 秒后在随机位置重生
- 接近玩家 1.5 单位时造成 20 HP 伤害 + 击退效果

**连击系统**：2 秒内连续命中的 combo 递增，分数乘以 `(1 + combo × 0.5)` 倍率。屏幕中央显示 "x2 COMBO!" 浮动文字。

**UI 层**：全部使用 HTML/CSS 覆盖在 3D 画面上方：
- 左上：HP 血量条（颜色随血量变化：绿→黄→红）
- 右上：分数 + 连击显示
- 顶部居中：60 秒倒计时（低于 10 秒红色闪烁）
- 右下：武器名 + 弹药信息
- 屏幕中央：CSS 绘制的十字准星（命中变红）

**CSS 准星**：使用 4 个绝对定位的 `div`（上下左右各一条短线）+ 中心红点组成，射击时通过 `translateY(-3px)` 模拟后坐力抖动。

## 六、跨游戏通用技术

### 6.1 响应式设计

所有游戏支持 PC 和移动端：
- Canvas 使用 `max-width: 100%` + `height: auto` 自动缩放
- 点击坐标通过 `getBoundingClientRect()` 和缩放比（`canvas.width / rect.width`）转换
- 同时绑定 `click` 和 `touchstart` 事件，触摸事件调用 `preventDefault()` 阻止默认滚动

### 6.2 音效体系

全部音效通过 Web Audio API 的 `OscillatorNode` 实时合成：

| 音效 | 频率 | 波形 | 持续时间 |
|------|------|------|----------|
| 点击 | 600-800Hz | square | 50-80ms |
| 消除/匹配 | 523→659→784Hz | sine | 150ms×3 |
| 胜利 | 523→659→784→1047Hz | sine | 250ms×4 |
| 失败 | 300→200Hz | sawtooth | 300-500ms |
| 射击 | 80-200Hz | sawtooth+square | 50-150ms |
| 命中 | 1000Hz | square | 80ms |

### 6.3 数据持久化

各游戏高分存储在浏览器 localStorage 中：

| Key | 游戏 |
|-----|------|
| `highscore_goose_game` | 抓大鹅 |
| `highscore_chinese_chess` | 中国象棋（胜局数） |
| `highscore_hide_seek` | 捉迷藏 |
| `highscore_spot_diff` | 找不同 |
| `highscore_match_game` | 消消乐 |
| `highscore_fps_game` | FPS 射击 |

### 7.5 各游戏操作指南

#### 🦆 抓大鹅
- **目标**：消除所有方块
- **操作**：点击未被遮挡的方块，移入底部槽位，3 个相同自动消除
- **道具**：移出（移除槽位末尾）、撤回（撤销上一步）、洗牌（重排剩余方块）
- **失败条件**：槽位满 7 个且无法消除

#### ♟️ 中国象棋
- **目标**：将死对方将/帅
- **操作**：点击己方棋子选中（显示可走位置），再点击目标位置移动
- **功能**：悔棋（撤销上一步）、新局（重新开始）
- **规则**：完整中国象棋规则，红方先行

#### 🔍 捉迷藏
- **目标**：在场景中找到所有隐藏物品
- **操作**：仔细观察画面，点击隐藏物品的位置
- **提示**：右侧面板列出需要寻找的物品清单
- **计分**：速度越快、错误点击越少，得分越高

#### 🧐 找不同
- **目标**：找出两张图片间的所有差异
- **操作**：点击任意一张图中的差异位置
- **提示**：点击"💡 提示"按钮高亮一处差异（每关 3 次）
- **计分**：找齐差异 + 剩余时间奖励

#### 🍬 开心消消乐
- **目标**：在 30 步内达到目标分数
- **操作**：点击一个方块选中，再点击相邻方块交换位置
- **规则**：交换后必须形成 3 连才有效，否则自动交换回来
- **连击**：连锁消除时分数倍率递增

#### 🔫 FPS 射击
- **目标**：60 秒内获得尽可能高的分数
- **操作**：
  - 点击画面锁定鼠标 → 移动鼠标控制视角
  - 左键射击 / R 键换弹
  - 数字键 1（手枪）2（步枪）3（霰弹枪）切换武器
  - ESC 解锁鼠标
- **注意**：避开骷髅靶标（黑色，命中扣 15 分），被敌人靠近会扣血

### 7.6 已知限制

1. **FPS 游戏不支持移动端**：依赖 Pointer Lock API 和鼠标控制，移动设备无法正常操作
2. **高分数据仅存于本地**：localStorage 数据绑定浏览器，清除缓存会丢失记录
3. **Three.js 需联网加载**：FPS 游戏首次打开需要从 unpkg CDN 下载 Three.js（约 600KB），之后浏览器会缓存
4. **捉迷藏/找不同的场景为程序绘制**：细节有限，非照片级画面

### 7.7 扩展建议

- **新增游戏**：在 `games/` 下创建新目录，放入 `index.html`，在主页 `views/index.ejs` 添加卡片即可
- **替换素材**：将图片放入 `assets/images/`，在游戏代码中用 `new Image()` 加载替代 Emoji/Canvas 绘制
- **添加后端功能**：在 `server.js` 中添加 API 路由，可实现在线排行榜、用户登录等功能
- **部署上线**：项目可直接部署到任何支持 Node.js 的平台（Vercel、Railway、自有服务器等）

```
│   ├── 受击: hp-1 + 闪红, hp=0 → 死亡 + 掉落弹药
│   ├── 接触: 距离<1.5 → 扣20HP + 击退双方
│   └── 重生: 死亡10秒后随机位置复活
│
├── UI 模块 (HTML/CSS 覆盖层)
│   ├── 左上: HP 血量条 (CSS 渐变色随血量变化)
│   ├── 右上: 分数 + 连击倍率
│   ├── 顶部: 60秒倒计时 (<10秒红色脉冲动画)
│   ├── 右下: 武器名 + 弹药 (弹夹/储备)
│   ├── 中央: CSS 十字准星 (4条线 + 红点)
│   ├── 受伤: 全屏红色闪烁 (opacity 0→0.3→0)
│   └── 开始/结束: 全屏遮罩弹窗
│
└── 游戏循环
    └── requestAnimationFrame → animate(time)
        ├── 计算 deltaTime (帧间时间差)
        ├── 更新计时器 (每秒-1)
        ├── 触发靶标波次 (每5秒)
        ├── 更新移动靶标位置
        ├── 更新敌人移动 + 碰撞
        ├── 更新弹药拾取 (旋转 + 超时移除)
        └── renderer.render(scene, camera)
```
